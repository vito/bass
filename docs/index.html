Bass is a scripting language for running commands and caching the shit out of them.

The plan is to support sophisticated CI/CD pipelines without adding a redundant declarative layer. Ultimately all these CI/CD systems boil down to running commands. Bass targets that layer of abstraction instead.

Bass's goal is to make shipping predictable, repeatable, and fun. If you'd like to try it out, grab the latest release and skim the guide.

Predictably, to build code just run whatever command you already use.

In this example we use -> to avoid deeply nested function calls and with-env to set CGO_ENABLED=0 so that we get a static binary.

(use (.git (linux/alpine/git)))

(defn go-build [src & args]
  (from (linux/golang)
    (cd src
      (-> ($ go build & $args)
          (with-env {:CGO_ENABLED "0"})))))

(let [src git:github/vito/booklit/ref/master/
      built (go-build src "./cmd/booklit")]
  (-> (from (linux/alpine)
        ($ built/booklit --version))
      (read :raw)
      next))
